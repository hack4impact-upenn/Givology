cygwin setup:

  1. select 'install from internet'
  2. root install directory should be something like C:\cygwin (but it doesn't matter much)
  3. select a place for it to download packages to temporarily (unimportant decision)
  4. internet connection type (probably just direct connection)
  5. select a place to download from arbitrarily
  6. select packages to install:
    1. in the upper right corner, click the 'view' button until it says 'Full' to the right, so that we can view all categories (probably just one click needed).
    2. might want to widen the box so we can see everything.
    3. you can select packages to install by clicking on where it says 'skip', so that it will select a version of that package. if it has a version number or says 'keep', then you already have that selected or installed. when clicking on a package, it will also select any dependencies it needs.
    4. the following packages are needed (if it says 'skip' on them, click that to change it):
      • 'gcc: C compiler upgrade helper'
      • 'ImageMagick: Image manipulation software suite (utilities)'
      • 'libsqlite3-devel: An embeddable SQL database engine (3.x development)'
      • 'openssh: The OpenSSH server and client programs'
      • 'openssl: The OpenSSL runtime environment'
      • 'python: An interactive object-oriented scripting language'
      • 'sqlite3: An embeddable SQL database engine (3.x apps and docs)'
      • 'subversion: A version control system'
      • 'wget: Utility to retrieve files from the WWW via HTTP and FTP'
  7. wait a while for it to download and install all those things (this could take as long as a half hour, depending on your internet connection and/or hard drive speeds)

setting up other things (pysqlite and django):

get and install django 0.96 ...

  1. `cd`
  2. `wget https://www.givology.org/static/Django-0.96.5.tar.gz`
  3. `tar xzvf Django-0.96.5.tar.gz`
  4. `cd Django-0.96.5`
  5. `python setup.py install`

and get and install pysqlite ...

  1. `cd`
  2. `wget https://www.givology.org/static/pysqlite-2.6.3.tar.gz`
  3. `tar xzvf pysqlite-2.6.3.tar.gz`
  4. `cd pysqlite-2.6.3`
  5. `python setup.py build_static install`

to check out the code and get started (once you have an account on xp-dev):

create an account on xp-dev. email me (cjmackey@stwing.upenn.edu) for information on how to get access to the svn repository.

  1. `cd /django` #(this step is not needed if you want to put the code elsewhere)
    #if you have checked out the code before, move the giv folder somewhere else, eg: `mv giv giv_old`
  2. `svn checkout http://svn.xp-dev.com/svn/cjmackey_giv/giv/` #(this may take a while, since there's a lot to download)
  3. `cd giv/proj`
  4. `python manage.py syncdb` #(choose a superuser username and password, doesn't need to be strong since it's only on your own computer)
  5. `python manage.py runserver` #(starts the server on your own computer, which you can visit in your web browser at http://localhost:8000/ )

to see other people's changes, update your working copy:

  1. `cd /django/giv` #(it's important that you go here, otherwise you might only update a part of your working copy)
  2. `svn update`

to commit your changes:

  1. `cd /django/giv` #(it's important that you go here, otherwise you might only commit a part of your working copy)
  2. `svn status`
    #(look at the list of files; M means modified, A means added, ? means it doesn't know)
  3. `svn add FILE` #(if you want to change a file from ? to A (if you made a new file that you want everyone else to see))
  4. `svn commit -m "YOUR COMMIT MESSAGE"` #(don't forget the minus m and commit message! briefly state what you changed.)
    #(you can also commit individual files, if you want, by running `svn commit FILE -m "YOUR COMMIT MESSAGE"`)
